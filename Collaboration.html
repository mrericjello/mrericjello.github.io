Forums

Q&A is more powerful than you think.

As a developer, you may not always know the answer to your problems in coding. Many times you have to dive into code and figure out what's wrong. But even then you may not be able to find the answer. Online Developer Forums are a way for developers to put their code out to the general web community and may get responses back that help them. However, this is not a tried and true way of solving coding issues. You may not get a response on the web until 3 months later! But if you put out specific questions, with access to your code, you increase your chances of getting a response. A couple of online dev forums are Desinger Hangout, Google Plus Threads, and #Front-End Developers.

Frameworks and Web Site Builders

Pre-Assembled Technology

One great thing about sharing ideas in the tech community is finding ways to make web technology faster. For example, if you're a developer and need to make a quick solution for a client that looks great without a lot of work, then you might want to consider a Framework. Frameworks are pre-assembled technologies (pre-written code) that developers can use as a template and manipulate it to make it their own. Frameworks like Bootstrap, Brackets, and Google Polymer have become a staple in making websites. Downloading these assembly technologies will make websites more efficient and take less time to make. The down side to this pre-written code is that you have to know how to manipulate it, and changing some elements or styles may not always be clear.

What if you're not a developer and have no idea how to write code? Or you're a developer that wants to make a simple intranet site for functions? Developers realized that not everyone wants to learn how to code or has the time to write out code. Another option to making websites easily are Web Site Builders. Web Site Builders are applications such as Google Sites, Adobe DreamWeaver, or Microsoft Visual Studio that help you build websites without even coding a single line. It's great for creating simple websites or apps without knowing how to code. The down side is Web Site Builders have large limitations on what they can do, including restricting writing in custom HTML, CSS, and JS.

Git and Git Hub

Repositories and Semantic Versioning

A repository is a place to store files, data, and historical revisions that can be managed. When developers make changes to their website, or software, they need to manage those files on the web. Repositories are great places to view, test, manage, find and share website or file data. Git is a widely used versioning software used for front-end and Git Hub is an online repository for files to be placed on the internet. The great part about these working together is that you may clone the online repository to your local OS system, and upload any revisions to your website up to Git Hub. Git Hub keeps track of all revisions with commit messages. So if anythign changes, you can also revert the changes back.

Hyper Text Markup Language 5

HTML5

HTML is the basic most coding language every Front-End developer must learns starting out. It is essentially structuring a document like in MS Word, but on the internet. You can add titles to the pages, headings, lists, body, text, etc. The current most version of HTML is HTML 5. The W3C made a standardiztion of HTML as the bases for all browsers to render web pages. Every webpage is made from HTML.

Cascading Style Sheets 3

CSS3

CSS is a Front-End language that styles certain portions of the HTML document. It specifies specific elements in the HTML code and tells how to specifically style the selected area with properties and values. It allows for modular site designs.

There are CSS pre-processors that have been created to fill the gap in making efficient code. Scripting language that extends the functionality of CSS and must be compiled into native CSS code before publishing. Preprocessor benefits are that they add additional functionality that CSS lacks, their syntax makes writing css more efficient, and many browser inconsistencies are dealt with automatically when the CSS is processed. To name a few are Sass writtten in ruby, Less in .js, and Stylus in .js. Preprocessor cons are that in team environments, if the production code is shared, everyone will need to know the preprocessor syntax. Processed CSS can be bloated, and add additional layer of complexity to workflows.

Javascript

.JS

Javascript is a scripting language that was developed in 1995 by Netscape to increase functionality on webpages. It allows the addition of interactive elements, updating data in the browser, can open new windows or give users instant feedback. It can update, sort and present data within the browser with no additional server requests.

JQuery

Java Script Library

Every developer should have an assembly technology in their toolkits in order to create solutions faster. Rather than continually remaking lines of code for different projects. The most popular used JavaScript Library that has become a standard of use is jQuery. jQuery is a broad, general purpose library that is used in a wide array of web pages for it's built-in codes. The downside to using jQuery is that it can increase the amount of serverside requests of loading your pages. Therefore slowing down your site loading speed.
